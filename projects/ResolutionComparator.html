I'm thinking of getting a new monitor, and I'm considering splashing the cash on a 4K monitor. As someone who has a retina-display mac, the biggest problem I have with using other monitors simulataneously is the difference in pixel density. So, as a simple way to calculate and compare the pixel densite of different monitors, I built this simple widget over a weekend.

I know, I'm sad. Deal with it.

There's nothing particularily funky going on. Just a function that calculates the pixel density (that could honestly do with some tests if I could be bothered): 

function calculate_ppi() {
  var horizontal = $(".calculation_input_horizontal").val();
  var vertical = $(".calculation_input_vertical").val();
  var diagonal = $(".calculation_input_diagonal").val();

  var aspect_ratio = horizontal / vertical;
  var angle = Math.atan(vertical/horizontal);

  var height = Math.sin(angle) * diagonal;
  var width = Math.cos(angle) * diagonal;
  var area = height * width;

  var num_of_pixels = horizontal * vertical;
  var pixels_per_square_inch = Math.sqrt(num_of_pixels / area );
  return( Math.round( pixels_per_square_inch * 100 ) / 100 );
}

We then just store that in the browser's local storage when the user hits save. Then display a DataTable from the stored data.

Oh, and then just to add a bit of responsiveness, the columns of the table vanish when the screen size is too small. Not the best for functionality but a simple hack to ensure it doesn't look like crap. 

@media(max-width: 900px) {
    table.dataTable thead tr th:nth-child(6), table.dataTable thead tr th:nth-child(7),
  table.dataTable tbody tr td:nth-child(6), table.dataTable tbody tr td:nth-child(7) {
    display: none;
  }
}

@media(max-width: 600px) {
  table.dataTable thead tr th:nth-child(2), table.dataTable thead tr th:nth-child(3), table.dataTable thead tr th:nth-child(4),
  table.dataTable tbody tr td:nth-child(2), table.dataTable tbody tr td:nth-child(3), table.dataTable tbody tr td:nth-child(4) {
    display: none;
  }
}
